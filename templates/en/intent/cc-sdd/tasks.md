# Tasks Hints (draft for cc-sdd)

> Updated by `/intent-export-cc-sdd`. This is not cc-sdd's tasks main body but check items (bullets) for when cc-sdd generates the tasks. The main body is generated by cc-sdd. The leading "Intent-derived constraints" are intended to be transcribed into each task so that an impl subagent in a different session can reference them via the cc-sdd deliverable (tasks.md).

## Source Packet

The target packet.

## Intent-derived constraints (premise of transcription into each task)

- **Parent Intent**: the L0/L1/L2/L3 this packet supports.
- **Invariants**: the constraints each task must not break (at a granularity referenceable per task).
- **Anti-direction**: a summary of the local optimizations / quick-fix refactors to avoid.

## Tasks check items

Perspectives to include when cc-sdd generates the tasks:

- [ ] Pre-check: confirm related intent / compass / invariants / existing behavior / scope of impact
- [ ] Characterization Tests: add tests that pin down existing behavior
- [ ] Migration Slice: create a small, rollbackable migration unit
- [ ] Implementation: implement
- [ ] Validation: tests / type checking / lint / manual check
- [ ] Rollback Check: confirm how to revert on failure
- [ ] Done Criteria: satisfies the parent intent, does not break invariants, and meets the packet's Expected Behavior

Leave, in each task's description, references to the corresponding parent intent and invariants.
