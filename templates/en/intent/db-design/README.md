# DB Design Draft Output Derived Views

> `/intent-db-design` reads the canonical intent (the Invariants/Anti-direction of `.intent/intent-compass.md`), the target packet, and existing schema/migrations read-only, then writes a **DB design draft** — table definitions, constraints, indexes, and naming — under this directory. Everything except this README is **NOT tracked by Git (local-only)**. The views placed here are **derived, NOT the source of truth**. The authority remains the canonical intent files; these views are merely a projection (snapshot) at generation time.

## Structure

```
.intent/db-design/
├── README.md                 # this description (tracked by Git)
└── <packet-slug>/
    └── db-design.md          # the DB design draft produced by /intent-db-design (untracked, regenerated by full replacement)
```

## Nature of this directory

- **Derived / regenerable**: `db-design.md` is regenerated by **full replacement** from the canonical intent, the target packet, and the existing schema every time `/intent-db-design` runs. Hand edits are overwritten on the next run (do not edit).
- **A design draft, not requirements**: What lives here is a **DB design draft**, not requirements. It is not mixed into the cc-sdd / OpenSpec export artifacts (requirements). It is not a finalized implementation spec; it is a foundation that keeps the DB description in the kiro design phase from being filled in thinly "from imagination."
- **Not the source of truth**: The authority for design intent stays in the canonical `.intent/intent-tree.md` / `.intent/intent-compass.md` and the packets. This view is a **reading** projecting the intent and existing schema; editing it here does not propagate to the canonical files or the existing schema. To change the design, return to the discover / compass / packets dialogue to update the intent, then run `/intent-db-design` again (there is no automated handoff from this view back to the canonical files).
- **Not tracked by Git**: `db-design.md` and everything else except this README are local-only and not tracked (only this README is tracked). By design this avoids merge conflicts and loss from overwrites in team settings.
- **Read-only projection layer + scoped Write**: `/intent-db-design` does not modify the canonical intent or the existing schema (read-only); its output writes are confined to this `.intent/db-design/` directory.
