# handoff 引き継ぎブリーフ（ローカル専用・使い捨て）

このディレクトリは **セッションを乗り換えるときの引き継ぎ文書の正式な派生保存先**です。生成は外部の `handoff-bridge` が担い、明示された未使用のファイルへだけ書き出します。保存先は通常、intent-planner 側の各スキルが `.intent/handoff/handoff-<日付>-<短い案件名>.md` の形で自動導出して渡します（利用者が別の保存先を明示したときだけそれを優先）。引き継ぎ文書は、次のセッションが正本を確認して最初の行動へ進むための短い文書です。

```
.intent/handoff/
  README.md              ← このファイル（追跡される・説明）
  <生成されたブリーフ>.md  ← 引き継ぎブリーフ（git 非追跡・使い捨て）
```

## なぜ git 非追跡なのか

ブリーフの価値の核は「**その瞬間のローカル事情**」です — 未コミットの変更・並行して動いている別セッションの注意点など、いま乗り換える人に要る情報。これらは git で追跡すると、すぐ古くなる情報がチームの共有物に残ってしまいます。だからブリーフ本体は**このディレクトリに置いたまま git 非追跡**にし（インストーラが `.gitignore` に追記します）、README だけを追跡します。

## ブリーフの性質（読むときの前提）

- **派生（derived）であって正本ではありません。** 冒頭に生成時点が書かれています。作業を再開するときは、ブリーフだけで完結させず、必ず正本（該当 packet・compass の関係する記号・plan）を pull してください。**食い違ったら正本が常に勝ちます。**
- **使い捨てです。** 失っても `.intent/` の状態から再生成できます。古くなったら消して構いません。
- **機微な内容は生のまま書きません。** 詳細は正本への参照（pull ポインタ）で示します。

## 使い方

- 生成: 互換確認済みの `handoff-bridge` skill が利用できる場合に、その skill を明示起動します。保存先は intent-planner 側が `.intent/handoff/handoff-<日付>-<短い案件名>.md` の形で未使用の名前を自動導出して渡すため、利用者が指定する必要はありません（別の保存先を明示したときだけそれを優先）。未導入・非互換・確認失敗の環境では、各スキルは生成案内を出しません。
- 受取: `source` と `read_for` を確認して locator の正本を読み、`authority` / `provenance` と照合します。source を利用できない場合は受取条件として明示し、生成済み文書を失敗扱いしません。
- 乗り換え時の促し: `/intent-packets` や `/intent-writeback` の完了報告が、工程の切れ目で「続けるか・乗り換えるか」の判断材料を添えることがあります（案内であって強制ではありません）。
