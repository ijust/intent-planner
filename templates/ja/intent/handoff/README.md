# handoff 引き継ぎブリーフ（ローカル専用・使い捨て）

このディレクトリは **セッションを乗り換えるときの引き継ぎブリーフ**の置き場です。生成の担い手は移行中です（外部ツール化 GO 2026-07-12・DR157。導入後に外部ツールがここへ派生生成します）。ブリーフは `.intent/` の現在の状態（active packet・plan・割当宣言・発行ディレクトリ・直近の記録）から作られる、次のセッションが着手できる短い文書です。

```
.intent/handoff/
  README.md              ← このファイル（追跡される・説明）
  <生成されたブリーフ>.md  ← 引き継ぎブリーフ（git 非追跡・使い捨て）
```

## なぜ git 非追跡なのか

ブリーフの価値の核は「**その瞬間のローカル事情**」です — 未コミットの変更・並行して動いている別セッションの注意点など、いま乗り換える人に要る情報。これらは git で追跡すると、すぐ古くなる情報がチームの共有物に残ってしまいます。だからブリーフ本体は**このディレクトリに置いたまま git 非追跡**にし（インストーラが `.gitignore` に追記します）、README だけを追跡します。

## ブリーフの性質（読むときの前提）

- **派生（derived）であって正本ではありません。** 冒頭に生成時点が書かれています。作業を再開するときは、ブリーフだけで完結させず、必ず正本（該当 packet・compass の関係する記号・plan）を pull してください。**食い違ったら正本が常に勝ちます。**
- **使い捨てです。** 失っても `.intent/` の状態から再生成できます。古くなったら消して構いません。
- **機微な内容は生のまま書きません。** 詳細は正本への参照（pull ポインタ）で示します。

## 使い方

- 生成: 外部ツール（導入後）が担います。未導入の環境では、各スキルは生成トリガの案内を出しません（縮退）。
- 乗り換え時の促し: `/intent-packets` や `/intent-writeback` の完了報告が、工程の切れ目で「続けるか・乗り換えるか」の判断材料を添えることがあります（案内であって強制ではありません）。
