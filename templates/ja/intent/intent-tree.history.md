# Intent Tree History（退避された履歴）

> `/intent-writeback`・`/intent-improve` が、完結した機能の **Impact Analysis・出荷済み L4・役目を終えた機能追記セクション**を、`intent-tree.md` 本体から**退避（move）**します。読み手は人間で、必要なときにだけ参照します。
>
> **これは退避先であって削除先ではありません。** 退避された内容は「なぜその設計にしたか」の監査証跡として grep できる形のまま残ります（immutability は git に委ねますが、現用の参照性はこのファイルで保ちます）。`intent-tree.md` 本体は現役の L0–L4 の骨格だけに保ち、pull 規律（全文を読まない）の前提＝本体が軽い、を回復します。
>
> **退避は move であって edit ではありません。** 退避するとき、本体にあった文面・番号・意味を**変えず**にこのファイルへ移します（「整理のついでに古い表現を直す」「重複を統合する」をしない＝現役 canonical の改変と履歴退避を混ぜない）。
>
> **退避規律（A19 の active 面/履歴分離と同型）**:
> - **何を**: 完結した機能の `## Impact Analysis（… が触れる既存境界）` セクション、`## L4 Candidate Packets` の「出荷済み（参照）」群、役目を終えた `## 機能追記: <slug>` セクション。
> - **いつ**: その機能の `/intent-writeback` 完了時（学びを canonical へ戻し終え、機能が一段落した節目）、または `/intent-improve` の再整合時。
> - **どう**: 本体から該当セクションを切り取り、このファイルの末尾へ frontmatter スキーマタグ `status: archived` を付けて追記する（下記記法）。本体側からは除去する（move）。
> - **退避先が無いとき**: このファイルが未作成・不在なら退避をスキップし作成を促す（履歴を消さず本体に残したまま告知する）。

## 退避記法（このファイルの末尾に追記する）

各退避ブロックは、退避元の見出しをそのまま保ち、直後に1行のメタ行を添える:

```
## 機能追記: <slug>（… / Mode: …）   ← 退避元の見出しをそのまま保つ
> status: archived | archived_at: <ISO 8601> | from: intent-tree.md
（以下、退避元の本文を文面・番号を変えずそのまま）
```

- `status: archived` で grep すれば退避済みだけを引ける（`intent-tree.md` 本体の現役セクションは `status: active` を付けるか、タグ不在＝現役とみなす後方互換読みで引ける）。
- 退避元の内容は behavior-preserving に保つ（番号・識別子・文面を変えない）。
