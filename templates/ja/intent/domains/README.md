# domains 領域ガバナンス（領域定義と owner 宣言）

このディレクトリは **compass/tree のガバナンスを領域単位に委譲する**ための置き場です（federated governance・INV101）。実体（単一の `.intent/`・grep 全域検索）は割りません。領域ごとに委譲するのは**所有権（誰が決め・書くか）と実行スコープ（どこまでを対象に improve/validate を回すか）だけ**です。

```
.intent/domains/
  README.md               ← このファイル（追跡される・領域定義もここに書く）
  owners/                 ← owner 宣言（git 非追跡・1宣言=1ファイル）
    <領域>-<session-rand>.md
```

## 領域定義（このファイルの下部・git 追跡・チーム共有）

領域は compass の各記号に付いた `[領域: <名前>]` タグ（INV47）の値です。**記号→領域の対応の正はタグだけ**で、この定義は領域名と一行説明の共有語彙にすぎません（宣言に記号を列挙しません＝DR193。同じ情報を二重に持たないので食い違いが構造的に起きません）。

同梱の初期領域は既存タグの値をそのまま収載したものです（新領域を発明しません＝Anti-543）。新しい領域が実際に必要になったときだけ足してください。

| 領域 | 一行説明 |
|---|---|
| always | 全領域に効く横断規律（複数領域にまたがる Invariant/Decision Rule）。領域スコープ実行でも必ず一緒に読む。 |
| 詰め | 意図の詰め方・発問・制約カタログ・質問パックに関わる規律。 |
| 派生 | 再生成可能な派生出力（overview・release-note・nl-spec・index 等）に関わる規律。 |
| 記録 | append-only 記録（deltas・export-log・drift-log・compass-archive）の分割・保管に関わる規律。 |
| 出口 | 下流 spec ツールへの受け渡し・射影（cc-sdd/openspec/speckit/to-spec）に関わる規律。 |
| 語彙 | 正規語彙（ubiquitous language）・造語検出・記号ラベルに関わる規律。 |
| 配布 | installer・templates パリティ・ルート規約文書・doc-sync に関わる規律。 |
| 検査 | intent-validate の検査軸・PBR 観点・オラクルに関わる規律。 |
| 並行 | 並行セッション協調・mode 状態のスコープ分離・領域ガバナンスに関わる規律。 |

領域を足すときは、この表に「領域名・一行説明」を1行足すだけです。owner や記号目録は書きません（owner は下記の宣言へ、記号はタグが正）。

## owner 宣言（`owners/` 配下・git 非追跡・ローカル専用）

「この領域は今このセッション/人が触っている」を宣言する置き場です。**書き込みを止める仕組みではありません**（read-only の助言・INV91）。他セッションが同じ領域を触ろうとしたとき、書き込み側（`/intent-writeback`・`/intent-compass`）が「この領域は別セッションが owner 宣言中」と一言気づかせるだけで、止めも奪いもしません。

owner は組織の情報なので **git 非追跡（ローカル専用）** です（DR192＝領域定義は共有するが owner は共有物に載せない）。並行セッションは別ファイルを持つので衝突しません（assignments 割当宣言と同型）。

ファイル名は `<領域>-<session-rand>.md`（`<session-rand>` は `[a-z0-9]` 4文字・シェルで生成し、frontmatter の `session:` にも同じ値を書く＝assignments 割当宣言と同型）。書き込み側（compass / writeback）が「自分のセッションの宣言か」を判定するときは、自分がその領域を触るときに置いた owner 宣言の `session`（＝自分の session-rand）と、読み取った宣言の `session` を突き合わせる（同じなら自分の宣言・警告しない）。中身は frontmatter の最小スキーマだけで足ります（それ以上を義務化しません＝宣言が重いと書かれなくなり使われなくなる・Anti-543）:

```markdown
---
domain: 並行                       # 領域名（上の領域定義の値）
owner: "本人"                      # 誰が持つか（自由記述・人/チーム名など）
session: a3f2                      # <session-rand>（このセッションを表す乱数4文字・ファイル名末尾と同じ）
declared_at: 2026-07-15T00:00:00Z  # 宣言日時（ISO 8601・シェルの date で取得）
---
```

- **領域定義に無い領域名の owner 宣言も置けます**（gate にしません）。整合の照合は書き込み側の気づき提示までで、拒否しません。
- **削除は人手**（宣言を機械が自動で消しません＝生きた宣言を消さないため・INV91）。触り終えたらセッション自身が削除します。

## 宣言が無いとき（恒久フォールバック・後方互換）

`.intent/domains/` が無い・空の repo では、領域ガバナンスは何も変わりません（従来動作・INV101/DR133 と同型の恒久フォールバック）。領域スコープ実行（improve/validate）も、領域が定まらなければ黙って全量読みにフォールバックします。owner 宣言も任意で、置かなくても全スキルが従来どおり動きます。
