# Context Cost Cues（コンテキストを食う場面の気づきカタログ）

> `/intent-discover`・`/intent-improve` が読みます（`drift-watch: on` のとき）。**コンテキストを食う場面に気づかせるための手がかりであり、否定・矯正はしません。** 大量スキル導入や全文投入は精度を上げる正当な選択でありえます。**どのログにも記録しません**（消費量は計測不能で、設定やスキル構成といったプライバシーにも踏み込みません）。利用者が自分の現場で踏んだ型を追記して育てるファイルです。

## このカタログの読み方

- **これは網羅ではありません。** ここに載っている型は出発点（seed 4件）にすぎません。あなたの現場で実際にコンテキストを食った場面を型として追記し、育てていく前提のファイルです。
- 各型は集計キー（`id`）で識別します。型を増やすほど、「コンテキストを食っているかもしれない」と気づける範囲が広がります。
- **これは規範ではなく気づきです。** 型に「当てはまった」ことは無駄の確定ではありません。大量スキルの導入・全文ロードは意図的で正当な選択でありえます。当てはまったら「これがコンテキストを食っているかもしれない」と**気づかせるだけ**で、無駄か否かの判断は利用者に委ねます。「直せ」とは言いません。
- **どのログにも記録しません。** 消費量は計測できず、何が文脈を食うかは人により正当に異なるため、記録すればプライバシーに踏み込みます。気づきは read-only の助言に留めます。

## 型の書き方

新しい型は次のスキーマで追記してください。`id` は kebab-case の集計キーとして一意にします。

```markdown
## id: <kebab-case 集計キー>
- name: <短い名前>
- symptom: <題材・進み方の手がかり。discover/improve が題材と照合する弱い手がかり>
- もし意図せず効いていれば:
  - 代わりに: <意図せず効いていた場合に思い出せる軽い代替（薄い入口 / JIT pull / 入力限定）。命令ではなく任意の選択肢>
```

- `symptom` は「これに当てはまったら必ず無駄」という強い判定条件ではなく、照合の手がかりとして書きます（弱い手がかりにすることで fixation を避けます）。
- 「もし意図せず効いていれば」は、その型が疑われたときに思い出せる**任意の軽い代替**です。命令形（「直せ」「やめろ」）は使わず、意図的な高コスト選択を否定しません。

## id: full-compass-load

- name: 判断基準の全文ロード
- symptom: 判断基準を1点だけ要するのに、intent-compass.md（判断基準の文書）を全文読んでいるかもしれない。1つの Invariant / Decision Rule を確かめたいだけの場面で、文書全体を文脈に載せている進み方。
- もし意図せず効いていれば:
  - 代わりに: 必要な Decision Rule / Invariant を見出し単位で JIT に pull する（薄い入口から該当節だけ読む）軽い代替もあります。全文が要る場面ならそのままで構いません。

## id: whole-tree-read

- name: 意図ツリーの全文読み
- symptom: L0〜L3 のうち1階層だけ要するのに、intent-tree.md を毎回全部読んでいるかもしれない。特定の成果や能力だけ参照したい場面で、ツリー全体を文脈に載せている進み方。
- もし意図せず効いていれば:
  - 代わりに: 必要な層だけを名指しで読む軽い代替もあります。固定ロード文脈へ全文を貼らない選び方です。全体俯瞰が要る場面ならそのままで構いません。

## id: steering-bloat

- name: 固定ロード文脈の肥大
- symptom: 生成した steering や参照ドキュメントが毎回ロードされる固定文脈を肥大させているかもしれない。責務を足すたびに常時ロード対象が増え、コンテキストが線形に膨らんでいく進み方。
- もし意図せず効いていれば:
  - 代わりに: 固定ロードは薄く保ち、詳細は参照先へ逃がして必要時だけ pull する軽い代替もあります。常時その文脈が要る確信があるならそのままで構いません。

## id: redundant-reread

- name: 不要な再読み込み
- symptom: 直前に読んだ、または編集したばかりのファイルを、内容が変わっていないのに再度読み込んでいるかもしれない。同じ内容を短い間隔で複数回コンテキストに載せている進み方。
- もし意図せず効いていれば:
  - 代わりに: 直前に読んだ/編集した内容はそのまま使い、再読み込みを省く軽い代替もあります。並行編集などで変わった可能性を確かめたい場面ならそのままで構いません。
