# Constraint Library（あなたが育てる制約の台帳）

> あなたが繰り返し使う制約（Anti-direction / Invariant）を、このプロジェクトの `.intent/` 内に貯めて再利用するための台帳です。`/intent-compass` が叩き台を提示するとき、同梱の `constraint-starters.md` と並べてここに貯めた制約も候補に出します。**編集は人が行い、skill は read のみで自動改変しません。** 採用した制約をここへ書き加えるのも人の手（または人の明示承認）で行います。

## この台帳の使い方

- **これはあなたが育てるファイルです。** 同じドメインの開発を繰り返すうちに「自分はいつもこれを Anti-direction / Invariant にしている」という制約が溜まります。それをここへ追記して再利用します。
- **このプロジェクト内に閉じます。** この台帳は当該 repo の `.intent/` 配下にのみ置かれ、プロジェクトをまたいで共有されることはありません（他に資産を持ち出せない案件でも、制約がプロジェクトの外へ漏れない設計です）。再インストールやバージョンアップでも、ここに書いた内容は上書きされず保持されます。
- **canonical（人が編集）です。** 提示 skill はこの台帳を read のみで参照し、自動では書き換えません。
- **これは静的な文書です。** 利用のたびに外部サービス・データベースへ問い合わせることはありません。

## 記入スキーマ

繰り返し使う制約を次のスキーマで追記してください。`id` は kebab-case として一意にします。

```markdown
## id: <kebab-case 識別キー>
- name: <短い名前>
- 領域: <code | non-code>
- 適合する状況: <どんな案件・題材のときこの制約を効かせたいか>
- 制約:
  - Anti-direction: <避けたい方向>
  - Invariant: <守りたい不変条件>
- 由来: <この制約をどこで学んだか・なぜ自分の定番にしているか（任意）>
```

- ここに貯めた制約は、`/intent-compass` が叩き台を提示するときに同梱カタログと並んで候補に出ます（提示は候補まで・採否は人が判断します）。
- 同梱の `constraint-starters.md`（開発者がメンテして配る定石）と、この `constraint-library.md`（あなたが育てる制約）は別ファイルです。
