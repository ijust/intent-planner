# assignments 割当宣言（ローカル専用）

このディレクトリは **並行実装でどの packet を誰（どのセッション/エージェント）が実装中か**の宣言を入れる置き場です。実装に着手するセッションが `.intent/packets/active/<packet_id>.md` の実装を始めるとき、ここへ**割当宣言ファイルを1つ新規作成**します。

```
.intent/assignments/
  README.md                        ← このファイル（追跡される・説明）
  <packet_id>-<session-rand>.md    ← 1宣言 = 1ファイル（git 非追跡）
  <別packet_id>-<session-rand>.md  ← 別の宣言（並行セッションでも衝突しない）
```

## なぜ「1宣言=1ファイル」に分けるのか（並行書き込みの衝突を構造で消す）

複数のエージェント/セッションで同じ repo を並行して回すとき、割当宣言そのものも並行して書かれます。単一の集約ファイル（`.intent/assignments.md` 1枚）に全員が追記すると追記の衝突が起き、packet 本文に `## Assignment` 節を足すと「同じ packet を2人が触る」＝**防ぎたい二重着手そのものがファイル衝突を起こす**自己矛盾になります。

そこで **packet（`.intent/packets/active/<packet_id>.md`）と同じやり方**で、宣言ごとに別ファイルへ分けます。各セッションは**自分のファイルを新規作成するだけ**（他のファイルを一切書き換えません）。別ファイルなので**並行書き込みが構造的に衝突しません**（ロック・排他機構を持ちません）。

## 宣言ファイルの名前とスキーマ

ファイル名は `<packet_id>-<session-rand>.md`:

- `<packet_id>`: 実装中の packet の ID（`.intent/packets/active/` のファイル名から取る）。
- `<session-rand>`: 半角英小文字と数字 `[a-z0-9]` 4文字（着手時にシェルで生成＝`LC_ALL=C tr -dc 'a-z0-9' </dev/urandom | head -c 4`）。並行衝突を防ぐ。**agent 名・プロセス ID・環境変数から導出しません**（そのセッションを表す乱数のみ）。

ファイルの中身は frontmatter の最小スキーマだけで足ります（本文は任意メモに使えます）:

```markdown
---
packet_id: pkt-20260704-...-elcc   # 実装中の packet の ID
declared_at: 2026-07-06T02:00:00Z  # 宣言日時（ISO 8601・シェルの date で取得）
session: a3f2                      # <session-rand>（このセッションを表す乱数4文字）
note: ""                          # 任意メモ（どの端末/文脈で実装中か等・空でよい）
---
```

## 二重着手の見え方（read-only・止めない）

`/intent-status` と `/intent-overview` がこのディレクトリを **read-only** で読み、次を導出します（詳細な読み手契約は `skills/CONTRACT.md`）:

- **割当済み/未割当の一覧**: どの packet に宣言が付いているか。
- **二重宣言の warn**: 同一 `packet_id` を持つ宣言ファイルが**2つ以上**あれば、二重着手として名指しします（**警告のみで停止・拒否しません**）。並行の運用判断は人に残します。
- **放置宣言の観測**: 宣言だけ残って実装が止まった packet は、「宣言日からの経過」を観測として示すに留めます（**経過日数などの機械閾値で自動判定・自動解放しません**）。

宣言が1件も無いリポでは、既存フローの出力は従来と変わりません（後方互換）。

## 宣言の作成・解除（手動・自動化しない）

- **作成**: 実装に着手するセッション自身が宣言ファイルを1つ作ります。
- **解除**: 実装が完了して packet を done→archive したら、そのセッションが宣言ファイルを削除します（自動掃除は持ちません）。

## 追跡・非追跡

- このディレクトリ自体（`README.md`）は git 追跡されます（説明のため）。
- 宣言ファイル（`<packet_id>-<session-rand>.md`）は **git 非追跡**です（ローカル協調データ・installer が `.gitignore` に登録）。
- 割当宣言は packet の `state`（`draft`/`ready`/`implementing`/… の値）とは**別のレイヤ**です。宣言は「誰が・いつ着手宣言したか」だけを持ち、packet の frontmatter（12キー固定）や state を書き換えません。

## 次にどの packet を実装するか迷ったら（工程計画の参照・任意）

複数の未実装 packet を並行で消化していくとき、「次にどれに着手するか」の参考として、`.intent/packets/plan.md` に「工程計画」節（人が宣言したグループ見出しと着手順）があればその推奨着手順を見てください。読み方は上から順で、`done`・依存（`depends_on`）が未解決・**他セッションが着手宣言済み（このディレクトリに宣言がある）**ものを飛ばし、最初に残ったものが次の着手候補です（並び順と依存が矛盾するときは依存が勝ちます＝DR139）。これはあくまで参考の案内で、推奨と違う packet に着手しても構いません（強制しません）。工程計画が無ければ従来どおり自由に選びます。この参照は割当の別レイヤ規律を変えません（宣言・推奨のいずれも state を書き換えません）。
