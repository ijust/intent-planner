# 割当ビュー（assignment view・指定があるときだけ生成する面）

並行実装の割当宣言（`.intent/assignments/`）を read-only に束ねて「どの packet を誰が実装中か・二重着手はどこか」を1枚に映し、`.intent/overview/assignment-view.md` へ派生出力する手順。`intent-overview` の Step 2 で、**利用者が割当（誰が何を実装中か）を求めたときだけ**適用する（無指定の既定実行では本面を生成せず、Output の誘導1行だけを出す）。二重着手を見つけて名指しするまでで、止めない・自動解放しない（C40/A52・INV66・DR99・DR92）。

## 適用条件（既定では走らない）

- 利用者が `/intent-overview` に**割当（誰が何を実装中か）を要求した**ときだけ生成する（自然言語トリガ「割当」「誰が実装中」「assignment」「二重着手」「着手宣言」等）。既定の俯瞰ビューには自動同梱しない（pull 規律・コンテキストコスト）。
- 無指定の既定実行では、本面を生成せず overview.md の誘導1行に留める。既定実行の出力・所要は従来と変わらない（behavior-preserving）。
- 割当宣言ゼロ（`.intent/assignments/` が無い、または `README.md` 以外の宣言ファイルが1件も無い）のときは停止せず「割当なし」を明示した空ビューを返す（エラーにしない・後方互換・推測で埋めない）。

## 手順

1. **割当宣言を read-only で読む**: `.intent/assignments/*.md`（`README.md` を除く）の各宣言ファイルから frontmatter の `packet_id` / `declared_at` / `session` / `note` を読む。宣言ファイル名のパターン `<packet_id>-<session-rand>.md` の `<packet_id>` 部と frontmatter の `packet_id` を突き合わせる（意味判断不要の単純なファイル名照合）。読み取りのみで、宣言の作成・削除・書き換えをしない。
2. **active packet と突き合わせる（別レイヤ・state を読み替えない）**: 各宣言の `packet_id` を `.intent/packets/active/*.md` の frontmatter（正本。`index.md` は再生成で直る派生キャッシュ）と照合し、「割当済み packet（宣言あり）」と「未割当 packet（宣言なし）」に分ける。割当宣言は packet の `state`（5値）とは**別レイヤ**であり、`state` を書き換えず・読み替えない（DR99）。宣言ありと `state` を組で示してよいが、導出は read-only。
3. **二重宣言（二重着手）を名指しする（warn のみ・止めない）**: 同一 `packet_id` を持つ宣言ファイルが**2つ以上**あれば、二重着手として名指しする（どの `session` が同じ packet を触っているか）。**警告のみで停止・拒否しない**（並行の運用判断は人に残す・INV66）。別 `packet_id` の宣言が並んでいるだけのときは二重宣言としない（混同しない）。
4. **放置宣言を観測として示す（機械閾値なし・自動解放しない）**: 宣言だけ残って進行が止まった packet は、`declared_at`（宣言日）からの経過を**観測として示すに留める**。経過日数などの機械閾値で自動判定・自動解放・自動警告をしない（INV2/INV66・Anti-direction 321）。
5. **archive 済み packet への宣言残存を観測として示す**: 宣言の `packet_id` が `.intent/packets/active/` に存在しない（`archive/` へ移った等）ときは「archive 済み packet への宣言が残存」を観測として示す（自動削除しない・実装完了後の宣言解除は宣言したセッションの手動操作）。
6. **`.intent/overview/assignment-view.md` へ全置換で書き込む**: 冒頭に生成時点を明記する（derived・再生成可能・手動再生成のみ）。書込みは `.intent/overview/` 配下限定（canonical へ書かない）。

## 出力の形

- 冒頭: 生成時点・読んだ素材（宣言ファイル件数・active packet 件数）・**派生であり正本ではない旨**。
- 本体: 「割当済み N 件 / 未割当 M 件」の一覧（割当済みは `packet_id` + `state` + 宣言した `session` + `note`）。二重宣言の節（同一 packet を触る `session` を名指し・warn）。放置宣言の節（宣言日からの経過を観測として）。archive 済み packet への宣言残存があれば別区分。
- 空のとき: **「割当なし」を明示する**（無理に一覧を作らない）。宣言ゼロの repo では既存の俯瞰フローと同じく従来どおり（後方互換）。
- 末尾: 派生・正本ではない旨の注記と、宣言の作成・解除は宣言したセッション自身の手動操作である旨（本面は一覧まで・自動化しない）。

## 規律（守ること）

- **read-only 導出・warn-only・止めない（INV66・最重要のオラクル）**: 二重着手は名指しするが gate 化しない（export・push を止めない）。宣言の作成・削除・自動掃除をしない。放置宣言を機械閾値で自動解放しない。
- **宣言と state は別レイヤ（DR99）**: `state`（draft/ready/implementing/verifying/done）を書き換えず・読み替えない。宣言は「誰が・いつ着手宣言したか」だけを持つ。
- **二重宣言の warn は実害・派生のズレは衝突でない（A31/INV38）**: 同一 packet への二重宣言（並行実装の二重着手）は実害として名指しするが、派生物（本ビュー）の再生成ズレは「衝突」語で不安にさせない（再生成は手動実行時のみ・DR92）。実体ファイル（宣言ファイル・packet 本文）の並行衝突と、再生成で直る派生のズレを混同しない。
- **機械閾値・スコアリングを持たない（INV2）**: 放置宣言の経過は観測として示すに留め、日数閾値で自動判定しない。
- **出所を辿れないエントリを出さない**: 全エントリが読んだ宣言ファイル・packet へ辿れる（捏造ゼロ・evidence-anchored と同線）。
- **canonical を書き換えない**: 書込みは `.intent/overview/assignment-view.md` のみ。二重着手を実際に解くのは人の判断（本面は一覧まで）。
