# 新メンバー向けオンボーディングビュー（newcomer onboarding・指定があるときだけ生成する面）

新規参加メンバーが短時間で意図の全体像を掴めるよう、散在する `.intent/` 成果物から5点構成の1枚を read-only に集約し、`.intent/overview/newcomer-onboarding.md` へ派生出力する手順。`intent-overview` の Step 2 で、**利用者が新メンバー向けの入口（オンボーディング）を求めたときだけ**適用する（無指定の既定実行では本面を生成せず、Output の誘導1行だけを出す）。読むための入口地図であり、対話フロー・チュートリアル・進捗の判定はしない（DR106・A54・C34/DR92）。

## 適用条件（既定では走らない）

- 利用者が `/intent-overview` に**新メンバー向けの入口を要求した**ときだけ生成する（自然言語トリガ「新メンバー向け」「オンボーディング」「newcomer」「新しく入った人に見せる」等）。既定の俯瞰ビューには自動同梱しない（pull 規律・コンテキストコスト）。
- 無指定の既定実行では、本面を生成せず overview.md の誘導1行に留める。既定実行の出力・所要は従来と変わらない（behavior-preserving）。
- 素材が薄い（glossary が空／不在・進行中 packet ゼロ・compass 未生成等）ときは停止せず、該当節に**「該当なし」を明示した縮退版**を返す（エラーにしない・推測で埋めない）。

## 手順（5点構成・DR106）

1. **① 目的・成功の要約（L0/L1）**: `.intent/intent-tree.md` の L0（North Star 相当の目的）と L1（成果・成功基準）を read-only で読み、新メンバーが最初に掴むべき「このプロジェクトは何のためにあり、何ができたら成功か」を短く要約する。canonical と inferred（Assumptions / Open Questions 由来）を区別したまま映す。
2. **② 主要な横断 Invariant の抜粋**: `.intent/intent-compass.md`（分割収納 `.intent/compass/` があればそちら）から、複数領域に効く横断的な判断基準を抜粋する。選定は `[領域: always]` タグ・見出し等の**既存の標識を手がかりに読解で行い**、機械スコアリング・件数閾値で選ばない（INV2）。各項目は id（INV/DR 等）と一行の平易な言い換えを組で示す。
3. **③ 進行中の作業単位と着手状況**: `.intent/packets/active/*.md` の frontmatter（正本。`index.md` は派生キャッシュ）から進行中の packet 一覧と `state` を read-only で映す。`.intent/assignments/` に割当宣言があれば「誰が着手中か」を併記する（宣言と `state` は別レイヤ・DR99・書き換えず読み替えない）。進行中 packet ゼロなら「該当なし」を明示する。
4. **④ 正規語彙の主要語**: `.intent/glossary.md` から、新メンバーが最初の会話・成果物読解でつまずきやすい主要語を抜粋する（正規語＋一行説明）。選定は読解で行い、全転記しない（入口に必要な語まで）。glossary が空・不在なら「該当なし（正規語彙は未整備）」を明示する。
5. **⑤ 読み順ガイド**: ①〜④ を読み終えた新メンバーが次に開くべきファイルを、実在確認した上で順序付きで案内する（例: intent-tree → compass の関係領域 → 自分が触る packet）。**参照先はすべて生成時に実在を確かめたファイル・節に限る**（dangling を作らない・実在しない参照を1つも出さない）。
6. **`.intent/overview/newcomer-onboarding.md` へ全置換で書き込む**: 冒頭に生成時点を明記する（derived・再生成可能・手動再生成のみ）。書込みは `.intent/overview/` 配下限定（canonical へ書かない）。

## 出力の形

- 冒頭: 生成時点・読んだ素材（tree / compass / active packets / assignments / glossary の在否）・**派生であり正本ではない旨**・想定読者（新規参加メンバー）。
- 本体: 5点構成の節（① 目的・成功の要約 / ② 主要な横断 Invariant / ③ 進行中の作業単位と着手状況 / ④ 正規語彙の主要語 / ⑤ 読み順ガイド）。素材が無い節は削らず「該当なし」を明示して残す（5節の骨格は縮退版でも保つ・どの素材が無いかが新メンバーにも分かる）。
- 末尾: 派生・正本ではない旨の注記と、「ここから先は正本（intent-tree / compass / 該当 packet）を直接読む」への橋渡し。

## 規律（守ること）

- **derived-only 契約の継承（C34・最重要のオラクル）**: 書込みは `.intent/overview/newcomer-onboarding.md` のみ。canonical な `.intent/*.md` は一切書き換えない（正本非接触・再生成可能・手動再生成のみ・DR92）。
- **dangling を作らない**: ⑤読み順ガイドを含む全参照先は、生成時に実在を確かめたファイル・節に限る。実在しない参照が1つでも混ざるなら、その参照を落とすか「未生成」と明示する（捏造ゼロ・evidence-anchored と同線）。
- **縮退しても止まらない**: 素材ゼロでもエラー・停止にせず、「該当なし」を明示した縮退版を出す（推測で埋めない・無理に項目を作らない）。
- **機械スコアリングを持たない（INV2）**: ②の Invariant 抜粋・④の主要語の選定は、タグ・見出しの既存標識を手がかりにした読解で行う。重要度スコア・件数閾値・埋め込み検索を入れない。
- **オンボーディングを工程化しない**: 対話フロー・チュートリアル・理解度チェックを持たない（読める1枚まで）。30分目安（A54/L1-④）は運用の観測であり、本面で計測・強制しない。
- **既存プリセット非接触（pure addition）**: 既存の派生ビュー（decision-inbox / roadmap-projection / assignment-view / mermaid-views 等）の rule 本文・出力仕様を変更しない。本面は読み手別ビューへの1種追加である（新スキル・新 canonical を作らない・Anti-direction 348）。
- **ズレは派生のズレ（A31/INV38）**: 生成後に正本が動いてビューが古くなっても「衝突」ではなく「再生成で直る派生のズレ」。再生成は手動実行時のみ（DR92）。
