# 読む範囲の統制（指定必須・未指定は推奨提案→確認・既定除外・範囲外を読まない）

`intent-from-code` skill が、既存コードから意図を逆抽出する前に「どこを読むか」を統制するための正本。SKILL.md は手順と報告形式のみを持ち、読む範囲の決め方・除外の既定・範囲逸脱の禁止は本ルールを参照する。観測は直接の Read / Glob / Grep、または対象プロジェクトですでに利用可能なローカル read-only 解析・索引で行い、対象コード・入力文書・canonical な `.intent/*.md` を一切変更しない（書き込みは `.intent/code-ingest/` 配下のみ）。この統制は巨大リポでのコンテキスト浪費と意図しない範囲逸脱を防ぐための load-bearing な規律であり、抽出の入口ゲートとして働く（INV65 の read-only・staging 限定の一部）。

## 任意のコード解析・索引の扱い（導入・必須化しない）

- 対象プロジェクトですでに利用可能なローカル read-only 解析・索引は、シンボル、import・参照関係、呼び出し経路、依存方向、影響候補などの構造把握と、直接読む箇所を絞るために任意利用してよい。
- 利用可能とは、対象プロジェクトでホストが提供し、かつ現在のスキル実行権限から呼び出せる状態をいう。インストール済み・登録済みでも、その権限から呼べなければ利用不能として扱う。
- 本スキルは解析ツールを導入・初期化・必須化・更新・索引同期・状態管理しない。解析が利用不能（不在・権限不足を含む）、未初期化、古い、または不足している場合は、停止せず直接のコード読解へ戻る。
- 対象範囲が小さい、読む箇所が既知、または解析結果の取得と読解が直接読解より重い場合は、解析を使わず直接のコード読解を選べる。
- 解析結果は観測の手掛かりであり、意図を確定する根拠そのものではない。意図候補は LLM が現在のコードを読み、ファイル・シンボル等の復元根拠と inferred（推測）標識を付けて起こす。
- 解析結果が確定した対象範囲の外を示しても、範囲を暗黙に広げない。範囲外の情報を意図抽出の入力に使わず、必要なら利用者へ範囲拡大を確認する。
- コードや解析結果を外部 API・外部サービスへ送信しない。

## 対象範囲の指定は必須（全リポ走査を既定にしない）

- 逆抽出は**対象範囲（ディレクトリまたはモジュール）の指定を必須**とする。範囲が確定するまで意図抽出へ進まない。
- 対象範囲が指定されずに実行されたときも、**全リポジトリ走査を既定にしない**。未指定を「全量を読む」へ倒さず、既定では全リポ走査を行わない。
- 未指定のときは、リポジトリ構造（トップレベルのディレクトリ構成・エントリポイント・主要モジュール等）から**推奨範囲を提案し、利用者の確認を求める**。利用者が確認・確定した範囲だけを読む。
- 推奨範囲の提案は仮説であり、利用者が確定するまで暫定として扱う。勝手に確定して読み始めない。

## 依存物・生成物は既定で除外する（除外リストの正本）

対象範囲の内側であっても、依存物・生成物は**既定で読み込み対象から除外**する。除外の具体要素はこのルール本文を単一の正本とし、以下を既定で読まない。

- 依存物のインストール先: `node_modules/` ・ `vendor/` ・ `.venv/` / `venv/` ・ `site-packages/` など、パッケージマネージャが取得した外部依存の展開先。
- ビルド生成物・成果物: `dist/` ・ `build/` ・ `out/` ・ `target/` ・ `.next/` ・ `coverage/` など、ソースから機械生成される出力。
- lock ファイル群: `package-lock.json` ・ `yarn.lock` ・ `pnpm-lock.yaml` ・ `poetry.lock` ・ `Cargo.lock` ・ `Gemfile.lock` など、依存解決の固定結果。
- 生成・キャッシュ物: minify 済み/バンドル済みの生成コード・自動生成コード・`.cache/` 等のキャッシュ。
- 版管理・環境メタ: `.git/` 等の版管理内部データ。

> これらは意図の源ではなく依存解決やビルドの結果であり、読んでも意図抽出の材料にならずコンテキストを浪費する。除外要素は必要に応じて本ルール本文へ追記する（design や SKILL 本文には列挙しない）。利用者が明示的に含めたい場合のみ、確認の上で例外的に読む。

## 確定した範囲外を読まない（範囲逸脱の禁止）

- 確定した対象範囲（および確認済みの推奨範囲）の**外のファイルは読まない**。範囲外のファイルを意図抽出の入力に**用いない**。
- 抽出の途中で範囲外への参照や import を見つけても、そこへ辿って範囲外のファイルを読み込まない。関連が疑われるときは、範囲外である旨を出力に明示し、範囲拡大の要否を利用者に委ねる（推測で範囲を広げない）。
- 範囲外のファイルは意図候補・不変条件候補・沈黙ギャップのいずれの入力にもしない。範囲外から起こした記述を出力へ混ぜない。

## 空・不在の扱い（止めない・推測で埋めない）

- 指定された対象範囲が空、または実在しないときは、**停止せず**「読める素材が無い」ことを明示する。
- このとき推測で候補を埋めず、空の候補一覧を返す（fail-open）。存在しないコードから意図をでっち上げない。
