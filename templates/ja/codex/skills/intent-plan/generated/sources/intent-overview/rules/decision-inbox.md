# 判断待ちインボックス（decision inbox・指定があるときだけ生成する面）

散在する「いま人の判断を待っている問い」を read-only に横断集約し、`.intent/overview/decision-inbox.md` へ派生出力する手順。`intent-overview` の Step 2 で、**利用者がインボックスを求めたときだけ**適用する（無指定の既定実行では本面を生成せず、Output の誘導1行だけを出す）。溜まった判断を一望して捌くための地図であり、判定・スコア化はしない（C34 の1面・DR92・INV62）。

## 適用条件（既定では走らない）

- 利用者が `/intent-overview` に**インボックス（判断待ち）を要求した**ときだけ生成する（自然言語トリガ「判断待ち」「decision inbox」「未回答の問いを集めて」等）。既定の俯瞰ビューには自動同梱しない（pull 規律・コンテキストコスト）。
- 無指定の既定実行では、本面を生成せず overview.md の誘導1行に留める。既定実行の出力・所要は従来と変わらない（behavior-preserving）。
- 素材ゼロ（未回答の問いも承認待ちの学びも警告も無い）のときは停止せず「判断待ちなし」を明示した空ビューを返す（エラーにしない・推測で埋めない）。

## 手順

1. **判断待ちの素材を3源から read-only で集める**（すべて読み取りのみ）:
   - **(a) 未回答の Open Questions**: `.intent/intent-tree.md` の `## Open Questions`、`.intent/intent-compass.md` の未確定項目、`.intent/packets/active/*.md` の `## Open Questions` を読み、まだ回答（確定・裁量委任）が書かれていない問いを拾う。回答済み・実装裁量に委任済みと明記された問いは**載せない**（過剰列挙の抑止）。
   - **(b) 承認待ちの delta（未昇格の学び）**: `.intent/deltas/`（分割形が正本・単一 `deltas.md` はミラー/legacy）を読み、canonical へ昇格（承認）されていない delta を拾う。昇格済み・却下済みと記録された delta は載せない。
   - **(c) 直近の警告**: 追跡ファイルに記録が残っている warn だけを拾う（`.intent/export-log`（分割形が正本）の preflight warn、`.intent/drift-log.md`（drift-watch が on のときのみ）等）。セッション出力にだけ出た揮発 warn は拾わない（残っていないものは観測できない）。
2. **各項目に出所リンクと「誰が答えるべきか」を付ける**:
   - **出所リンク**: どのファイルのどの問い/どの delta か（ファイル名 + 見出し/該当行の目印）へ辿れる形にする。読み手がインボックスから「答える場所」へ直接移動できること（fit-criterion の柱）。
   - **誰が答えるべきか**: レンズ（観点代行）の記録が当該の問いに紐づいているときだけ、その**観点名**を添える（`ロールレンズ` の自由記述をそのまま使う・固定の職種3値を前提にしない）。記録が無ければ空欄にし、職種を推測で埋めない。組織の在否情報（誰が居ない・誰の担当か）は転記しない（INV60・観点名まで）。
3. **`.intent/overview/decision-inbox.md` へ全置換で書き込む**: 冒頭に生成時点を明記する（derived・再生成可能・手動再生成のみ）。書込みは `.intent/overview/` 配下限定（canonical へ書かない）。

## 出力の形

- 冒頭: 生成時点・読んだ素材（tree/compass/packets の Open Questions・deltas の未昇格分・追跡された warn の有無）・**派生であり正本ではない旨**。
- 本体: 「判断待ち N 件」の一覧。各行 = `[観点名（あれば）] 問い/学び/警告の一行要約 → 答える場所（出所リンク）`。源（(a) 問い / (b) 承認待ち / (c) 警告）で束ねて並べてよい。
- 空のとき: **「判断待ちなし」を明示する**（無理に項目を作らない）。
- 末尾: 派生・正本ではない旨の注記と、答え方の案内（問いは該当ファイルの編集、承認待ち delta は writeback の承認フロー、警告は該当スキルの再実行——回答 UI は本面に持たせない）。

## 規律（守ること）

- **回答は本面でしない（read-only・一方向）**: インボックスは「どこに何の判断待ちがあるか」を映すまで。回答・承認・警告解消は既存の各ファイル編集・承認フローで行う（本面に回答 UI を持たせない）。
- **過剰列挙・取りこぼしのどちらもしない**: 回答済み・裁量委任済みの問い、昇格済み・却下済みの delta は載せない（過剰列挙の抑止）。未回答の問い・未昇格の delta・残っている warn は取りこぼさない（見逃しの抑止）。
- **PM ツール化しない（INV62）**: 期限・担当ガント・進捗％・ベロシティを付けない。判断待ちの一覧と「答える場所」まで。
- **機微を転記しない（INV60）**: 「誰が答えるべきか」は観点名まで。組織の在否・個人の担当割り・実測の生データは載せない。
- **出所を辿れないエントリを出さない**: 全エントリが読んだファイルへ辿れる（捏造ゼロ・evidence-anchored と同線）。
- **ズレは派生のズレ（A31/INV38）**: 生成後に正本が動いてビューが古くなっても「衝突」ではなく「再生成で直る派生のズレ」。再生成は手動実行時のみ（DR92）。
- **canonical を書き換えない**: 書込みは `.intent/overview/decision-inbox.md` のみ。気づいた判断待ちを実際に片づけるのは人の判断（本面は一覧まで）。
