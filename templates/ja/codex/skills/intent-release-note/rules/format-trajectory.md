# format 写像：trajectory（変遷ノート）

`intent-release-note` skill が、`.intent/` 配下の変更履歴と意図照合の結果を、**「いつ・何を決めた・何を変えたか」を時系列で辿り直すための内向きの1枚**（変遷ノート）へ組み立てるための出力構造の定義。読み手は**将来の自分**（何日か空けて考えごとを再開する人）であり、既存の `format-changelog.md`（開発者向け・種類別）／`format-github-releases.md`（物語＋一覧）／`format-changelog-customer.md`（顧客向け・利用者影響先行）／`format-pr-description.md`（PR 説明の下書き）と並ぶ第5の出力形。

## 責務の境界（出力構造の定義であって読み取り・照合ではない）

- 本ルールは **format 写像（出力構造の定義）** である。変遷ノートのセクション構成・並び順・各項目の書き出しだけを定める。
- git log を読む・コミットを意図（packet name / parent intent / deltas）と照合する・range を解釈するといった手続きは **SKILL.md の責務**であり、本ルールは行わない。

## 素材の絞り込み（`.intent/` の変更履歴が主素材・DR117）

- この format の主素材は、照合済み素材のうち **`.intent/` 配下（intent-compass・intent-tree・packets・deltas 等）を変更したコミット**である。**アプリケーションコードの diff は読まない**（コード差分の読解・要約は既存 changelog 系 format の領分・DR117）。
- Intent trailer 付きコミットは packet との対応の**補助照合**に使う（記録に基づく対応と推測による対応の区別は SKILL の既存規律のまま）。

## 出力先（既存の変更履歴を上書きしない）

- 出力は `.intent/release-note/trajectory.md` へ**全置換**で書く（派生・再生成可能）。
- 既定の `release-note.md`（changelog 系 format の出力先）とはファイルを分け、変遷ノートの生成が変更履歴の生成物を上書きして消さないようにする。

## 読み手と設計制約（将来の自分・人間向けの書きぶり）

読み手は**中断をはさんで考えごとを再開する将来の自分**を想定する。この読み手は「先週の自分はなぜこう決めたか」を最短で思い出したい。

- **時系列・新しい順**: 直近の決定から過去へ遡る並びにする（再開時にまず直近の文脈が要るため）。
- **コミット羅列にしない**: 同種の変更（同じ packet・同じ決定に属する複数コミット）は1項目に束ねる。
- **辿れる参照**: 各項目に、根拠へ辿れる参照（コミット・packet 名・Invariant / Decision Rule の番号）を添える。

## 構成（3部・上から下へ）

| 順 | セクション | 何を置くか |
|---|---|---|
| 1 | **対象範囲** | 対象 range と format（trajectory）の明示。fallback したならその注記 |
| 2 | **変遷（新しい順）** | 「日付: 何を決めた・何を変えた」の時系列。各項目に辿れる参照を添え、理由が記録から辿れるものは「なぜ」を1行で添える。**理由が辿れない項目は「理由の記録なし」と明示する**（推測で物語化しない） |
| 3 | **意図と紐づかない変更** | 照合できなかった `.intent/` 変更コミットの薄い行（黙って捨てない） |

- 該当素材が無いセクションは「無い」ことを一行で明示する（黙って見出しごと消して読み手に探させない）。

## 理由の非捏造（最重要）

- 履歴・delta の記録から理由が辿れない変更に、**推測の動機・後付けの物語を書かない**。「理由の記録なし」と明示して並べる（履歴に無い理由を語らない）。
- 推測を書く場合は inferred（推測）標識を付け、記録に基づく「なぜ」と混ぜない。

## 機密の線引き（INV67・継承）

- 内向きの文書だが、外へ貼られうる前提で書く。載せるのは**要約+識別子**までとし、packet 本文・compass 本文の段落をそのまま転記しない。

## 異常系（エラーにしない範囲と fail-fast）

- `.intent/` の変更履歴が1件も無い（初期化直後の）repo では生成せず、その旨を告げる（fail-fast・空のノートを作らない）。
- range にコミットが無いときは SKILL の既定（空である旨の明示）に従う。

## 不変条件

- git・canonical を読み直さない・変更しない（読み取り・照合は SKILL.md、書き込みは派生ディレクトリへの SKILL.md の Write の責務）。
- アプリケーションコードの diff は読まない（DR117）。
- 素材に無い理由を体裁のためにでっち上げない（「理由の記録なし」の明示が既定・推測には inferred 標識）。
- 既存4 format（changelog / github-releases / changelog-customer / pr-description）の出力構造・出力先に触れない（純追加）。
