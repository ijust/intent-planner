# 読み手別の語彙開き：顧客向け出力では記号・内輪語を読み手の言葉へ（読み手分岐）

`intent-to-spec` skill が、生成する読める Spec を**コードを書かない読み手（顧客・経営・営業・非開発の当事者）**へ向けるとき、本文に残る内部記号・内輪の識別子を読み手の言葉へ開くための正本。各 format 写像ルール（意思決定メモ・上流向け・統合仕様書・定例報告・一枚もの）は、読み手が非当事者と判別されるとき本ルールを参照する。SKILL.md は「どの読み手にこの規律を適用するか」の振り分けだけを持ち、開き方の本体は本ルールに一本化する（各 format ルールへ同じ規律を重複させない）。

これは C14（記号の自然言語化）の思想を `intent-to-spec` の**出力面**へ適用したものである。C14 本体が「intent 成果物を書くとき裸記号にラベルを添える／ツール自身の配布物を publish 時にラベル置換する（shipped-internal-symbols）」＝**ツール自身のファイル**を対象にするのに対し、本ルールは「to-spec が利用者の案件向けに生成する外向き成果物」を対象にする（対象と発火点が別。機能・テストは二重化しない）。

## 読み手の判別（当事者か・非当事者か）

- **非当事者（顧客等）**: コードや `.intent/` を読まない読み手。既定でこの読み手を想定する体裁は一枚もの（stakeholder-onepager）。他の体裁でも、利用者が読み手を「顧客向け」等と指定したとき、または生成を顧客提出用と明示したときは非当事者として扱う。
- **当事者（repo の中の人）**: intent 成果物を辿れる読み手。既定でこの読み手を想定する体裁は意思決定メモ（読み手＝数週間後の自分）・上流向け・統合仕様書・定例報告（開発者向けの報告）。
- 判別は既存の source scope 解釈（引数・体裁種別）から導く。新しい状態機構・読み手の永続記録は足さない。体裁種別だけで割れる（顧客と当事者が混在しうる）ときは、利用者の読み手指定を優先し、指定が無ければその体裁の既定の読み手に倒す。

## 開き方（非当事者のときだけ適用する）

1. **記号は台帳を引いて人間語へ言い換える。**
   - 内部記号（`INV`＋番号・`DR`＋番号）は、記号→ラベル台帳 `scripts/symbol-labels.json` を引いて、配布言語（ja テンプレは ja 短名・en テンプレは en 短名）の短名へ言い換える（例:「（INV47）」→「（compass を領域タグで部分ロード）」相当）。
   - 台帳は shipped-internal-symbols（配布物の publish 時ラベル置換）が持つ既存のものを**流用**する。to-spec 側に第2の言い換え台帳を作らない（二重メンテ・同じ記号のラベルのずれを避ける）。
   - ラベルの正本は記号の定義行（DR22）であり、台帳はその短名を持つ。参照側で自己流のラベルを当てない。

2. **台帳に無い識別子は本文から省く。**
   - packet id（`pkt-…`）・スキル名（`from-spec`／`discover`／`intent-compass` 等）・実装語（`byte 不変`／`fail-fast`／`golden` 等）のように、人間語の短名が台帳に無いものは本文から**省く**。
   - 省くのであって、それらしい造語で埋めない（読み手が知らない語を勝手に作らない＝捏造抑制。素材に書かれた分だけを写す to-spec の性質に従う）。省いた結果、文が「どの決定・どの作業単位か」を指せなくなるなら、識別子でなく普通の言葉で対象を言い表す（例:「pkt-…-qo0h の入口拡張」→「断片メモの取り込み」）。

3. **当事者のときは従来どおり。**
   - 読み手が当事者と判別されるときは、記号・識別子を辿るための参照として従来どおり残す（消すとトレーサビリティを損なう）。この場合、本ルールの言い換え・省略は適用しない。

## 不変条件

- 開くのは非当事者向け出力だけである（当事者向け出力の記号・識別子は従来どおり残す）。
- 記号の言い換えは既存台帳 `scripts/symbol-labels.json` を引く（to-spec 専用の第2台帳を作らない）。
- 台帳に短名の無い識別子は省く（造語で埋めない）。
- 言い換え・省略は本文の読みやすさのための語彙操作であって、射影元の不変則・制約の内容を変えない（意味を落とさない・緩めない・強めない。不変則保持は捏造抑制ルールの責務のまま）。
- 読み手の判別に新しい状態機構・永続記録・機械検査を足さない（既存の source scope 解釈に相乗りする）。
