# format 写像：意思決定メモ（選択肢×基準の比較・採否先行）

`intent-to-spec` skill が、source scope 解釈が読み取った三層の素材を、**意思決定メモ**（迷っている・迷っていた選択を「選択肢×判断基準の比較」と「採用・不採用と理由」で読める1枚）へ組み立てるための写像規則。SKILL.md は手順と報告形式のみを持ち、「どの素材をどの順で・どの見出しに配置するか」は本ルールを参照する。

## 呼び出しの名前

- format 名トークンは `decision-memo`。日本語の呼び名（「意思決定メモ」「迷いごと比較シート」）でも同じ format を指す。出力先は `.intent/nl-spec/decision-memo.md`。

## 責務の境界（写像であって取材・評価ではない）

- 本ルールは **format 写像** である。素材の構成・順序・見出しへの割り当てだけを担う。
- 射影元（intent-tree / compass / packets / steering）を**読み直さない**。独自のパーサ・スキーマを持たず、source scope 解釈が受け渡した素材（どの層・どの見出し・どの packet 由来かを保ったもの）をそのまま並べ替える。
- **素材は既存の intent 成果物に閉じる（対話取材を足さない）**: 使うのは compass の Decision Rules（Context / Decision / Why / Alternatives considered / Consequences / Revisit when）と Open Questions、比較の過程が残る Evidence、packets の Deferred（切り出し）と Decisions の未定スロット。生成の途中で利用者へ聞き取りした新情報を本文へ直書きしない。まだ compass に書かれていない迷いは本 format の守備範囲外——「先に `/intent-discover` か `/intent-compass` で迷いを書き、それから再実行する」と案内して**生成しない**（fail-fast）。
- **評価・採否を発明しない**: 選択肢×基準の比較・採否・理由は、素材に**書かれている分だけ**を写す。表のセルを埋めるために書かれていない評価を補完しない（記録が無い欄は「記録なし」と明示する）。各記述のトレース付与・inferred 標識・不変則保持は**捏造抑制ルールの責務**であり、本ルールは行わない。

## 読み手と設計制約（採否先行）

読み手は「決めた本人（数週間後の自分）」とその同僚を想定する。決めごとの1枚は、経緯ではなく**採否（結論）を最初に**知りたい（定石 `bluf-message`）。

- **採否を冒頭に置く。** 何をどう決めたか（未決なら未決）を最初の節で言い切り、比較・経緯はその後に置く。
- **不採用の選択肢と理由を、採用と同じ紙面に残す。** 後日「やっぱりあの案は…」と蒸し返したとき、同じ1枚の中で「なぜ採らなかったか」まで読めることが本 format の目的。素材にある限り省かない。
- **1出力1目的（種別分離）。** 意思決定の記録に徹し、進捗報告・仕様詳細・手順書を同じ出力に混ぜない（定石 `doc-type-separation`）。
- **識別子はトレースとして添えてよい（当事者向けのとき）。** 既定の読み手が repo の当事者なので、判断の番号・packet 名は辿るための参照として有用。ただし初出には普通の言葉の言い換えを併記する（識別子だけの行にしない）。**読み手が非当事者（顧客等）と判別されるときは `rules/reader-vocabulary.md` に従い、記号（INV/DR）を台帳で言い換え・台帳に無い識別子（packet id・スキル名等）を本文から省く**（開き方の本体は同ルールに一本化し、ここでは重複させない）。

## 構成（上から下へ：採否 → 比較 → 不採用の理由 → 見直し条件）

source scope 解釈が受け渡した素材を、次の順で配置する。

| 順 | セクション | 何を置くか | 主な素材の由来 |
|---|---|---|---|
| 1 | 結論（決めたこと・未決のこと） | 対象の迷いごとに採否を1行ずつ言い切る。未決は「未決」と明示する | compass の Decision Rules（Decision）＋ Open Questions |
| 2 | 選択肢×判断基準の比較 | 迷いごとに選択肢を並べ、どの基準（Why / Criteria）でどう比べたか | Decision Rules（Context / Why / Alternatives considered）＋ 比較の過程が残る Evidence |
| 3 | 不採用の理由 | 採らなかった選択肢と、その不採用理由 | Decision Rules（Alternatives considered） |
| 4 | 見直し条件 | どうなったらこの決定を見直すか | Decision Rules（Revisit when） |
| 5 | まだ決めていないこと（あれば） | 未決の迷いと、次にどこで決める予定か | Open Questions ＋ packets の Decisions（未定スロット）・Deferred |
| 6 | 前提・未確定（あれば） | 確定した上記と**混ぜずに**別枠で置く前提 | intent-tree の Assumptions（inferred 由来） |

- セクション 1（採否）を必ず冒頭に置く。比較（2）から始めない（採否先行の核）。
- 比較は Markdown の表で書いてよいが、素材が表になるほど無い迷いは箇条書きで書く（形式のために内容を薄めない・セルを埋めるための補完をしない）。
- 該当する素材が無いセクションは「記録なし」と明示して省く（推測で盛らない）。
- 範囲内に迷い・判断の記述（Decision Rules / Open Questions / 未定スロット）が**ひとつも無い**ときは生成しない（fail-fast。上記「責務の境界」の案内を出して止まる）。
- セクション 6 は inferred 由来を canonical の本文へ溶かさず末尾の別枠に置く（素材が無ければ省く）。

## 不変条件

- 射影元を読み直さない・変更しない（写像のみ）。
- 採否（結論）を冒頭に置く順序を崩さない。
- 素材に無い評価・採否・理由を、表のセルを埋めるために足さない（記録が無い欄は「記録なし」）。
- 対話取材を足さない（素材は既存の intent 成果物に閉じる）。
- 不採用の選択肢と理由を省かない（素材にある限り必ず載せる）。
- canonical 由来と inferred 由来を混在させない（標識は捏造抑制ルールに委ねる）。
