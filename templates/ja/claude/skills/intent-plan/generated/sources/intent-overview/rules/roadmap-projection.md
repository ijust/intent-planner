# ロードマップ射影（roadmap projection・指定があるときだけ生成する面）

active packets の順序・状態・依存を read-only に束ねて「前向きの作業順序」を1枚に射影し、`.intent/overview/roadmap-projection.md` へ派生出力する手順。`intent-overview` の Step 2 で、**利用者がロードマップを求めたときだけ**適用する（無指定の既定実行では本面を生成せず、Output の誘導1行だけを出す）。**日付・期限・ガント・ベロシティは持たない**（順序と状態まで＝C34 の1面・DR92・INV62）。

## 適用条件（既定では走らない）

- 利用者が `/intent-overview` に**ロードマップを要求した**ときだけ生成する（自然言語トリガ「ロードマップ」「roadmap」「作業順序」「何が何を塞いでいるか」等）。既定の俯瞰ビューには自動同梱しない（pull 規律・コンテキストコスト）。
- 無指定の既定実行では、本面を生成せず overview.md の誘導1行に留める。既定実行の出力・所要は従来と変わらない（behavior-preserving）。
- packets ゼロ（`.intent/packets/active/` が空）のときは停止せず「packet なし」を明示した空ビューを返す（エラーにしない・推測で埋めない）。

## 手順

1. **active packets を read-only で読む**: `.intent/packets/index.md`（再生成で直る派生キャッシュ）ではなく、各 `.intent/packets/active/*.md` の frontmatter を正本として読む（`packet_id` / `name` / `state` / `updated_at` / `depends_on`）。`state` は6値（draft/ready/verifying/done + 旧値 active 等）を read-only で映すのみで、状態を算出・推論しない。
2. **順序と依存を束ねる（日付なし）**:
   - **順序**: `depends_on` の依存関係から「先に要るもの → それを待つもの」の前後関係を組み、日付ではなく**前後関係**として並べる。カレンダー上の予定・期限は付けない（INV62）。
   - **ブロッカー連鎖**: どの packet が何を塞いでいるか（A が未完なら B が始められない）を depends_on から辿り、連鎖として明示する。循環・未解決依存（active に無い packet への依存等）があればそのまま明示する（潰さない）。
   - **保留（parked）の区分**: 保留と記された packet（`state` や本文で parked/保留と明示されたもの）を別区分にまとめ、前向きの順序と混ぜない。
   - **工程計画のグループのまとまり（あれば反映・無ければ従来どおり）**: `.intent/packets/plan.md` に「工程計画」節（人が宣言したグループ見出しと着手順）があれば、そのグループのまとまり（見出し）でロードマップを束ね、グループ内は工程計画の並び順で並べる（DR139 の導出規則を共有し、ここで独自の順序解釈を発明しない）。節が無い・空なら従来どおり `depends_on` の前後関係だけで並べる。日付・スコアは持ち込まない（INV62/INV81・グループ見出しは人の語をそのまま使い固定語彙化しない＝Anti 432）。
   - **ジャーニーの束ね（あれば反映・無ければ従来どおり）**: ジャーニー正本 `.intent/packets/journeys/*.md`（README.md・archive/ を除く。読み手契約は skills/CONTRACT.md「ジャーニーの読み取り契約」）があれば、各ジャーニーの `packets` 列挙で構成 packet をジャーニー単位でも束ねて示せる。進捗は構成 packet の `state` から毎回導出し、ジャーニー側へ書き戻さない（DR200）。`lifecycle: archived` のジャーニーは束ねに出さない。列挙の packet_id が active に見つからないときは「見つからない」と明示して飛ばす（推測で紐づけない）。`journeys/` が無い・空なら従来どおり（恒久フォールバック・INV103）。既存の工程計画の束ね・既定の表示は変えない（純粋な追加）。
3. **進行実績と横断の束ねを添える**:
   - **進行実績**: `state` と `updated_at` から「最近 done / verifying になったもの」を読み取り、直近の進みを添える（日付列としてではなく「最近進んだ」の目印として）。
   - **リスクの集約**: packet 本文に `## リスク` 等のリスク欄が書かれていれば集約して束ねる。無ければ「未記入」と明示する（推測で埋めない）。
   - **体験段階の束ね**: 体験段階（experience stage）が書かれた packet があれば段階ごとに束ねる。無ければその束ねを省略し理由（未記入）を明示する（後方互換）。
4. **`.intent/overview/roadmap-projection.md` へ全置換で書き込む**: 冒頭に生成時点を明記する（derived・再生成可能・手動再生成のみ）。書込みは `.intent/overview/` 配下限定（canonical へ書かない）。

## 出力の形

- 冒頭: 生成時点・読んだ素材（active packets 件数・depends_on の有無）・**派生であり正本ではない旨**。
- 本体: 前向きの順序（前後関係）と各 packet の `state`。ブロッカー連鎖の節（何が何を塞ぐか）。保留（parked）の区分。最近進んだものの目印。リスク欄がある packet の集約。体験段階が書かれていれば段階ごとの束ね。
- 空のとき: **「packet なし」を明示する**（無理に順序を作らない）。旧 packet（`depends_on` なし・新欄なし）だけの repo でも「依存なし」「未記入」明示のまま成立させる（後方互換の読み取り契約）。
- 末尾: 派生・正本ではない旨の注記。

## 規律（守ること）

- **日付・進捗％・ベロシティを持たない（INV62・最重要のオラクル）**: 出力に日付列・締切・ガント・進捗％・ベロシティ（速度）を一切出さない。前向きの順序（前後関係）と状態・ブロッカー連鎖・保留区分・「最近進んだ」の目印までに留める。カレンダー上の予定表にしない。
- **状態を算出・推論しない**: `state` は frontmatter の値を read-only で映すだけ。進捗を単一％に潰さない・完了予測をしない。
- **index.md でなく frontmatter を正本に読む**: `packets/index.md` は再生成で直る派生キャッシュ。ズレは「衝突」ではなく「再生成で直る派生のズレ」として淡々と扱う（A31/INV38・直し方は `/intent-packets` 再生成を添える）。
- **後方互換の読み取り契約**: `depends_on` 不在は「依存なし」、リスク欄・体験段階が無ければ「未記入」と明示する。新欄の**記入**はしない（あれば束ねる・なければ未記入と明示するまで）。
- **出所を辿れないエントリを出さない**: 全エントリが読んだ packet へ辿れる（捏造ゼロ・evidence-anchored と同線）。
- **canonical を書き換えない**: 書込みは `.intent/overview/roadmap-projection.md` のみ。順序を見て実際に着手順を変えるのは人の判断（本面は射影まで）。
