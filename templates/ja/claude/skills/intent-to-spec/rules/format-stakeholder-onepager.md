# format 写像：ステークホルダー一枚もの（結論先行）

`intent-to-spec` skill が、source scope 解釈が読み取った三層の素材を、**ステークホルダー向けの一枚もの**（経営・営業・顧客窓口など、コードを書かない意思決定者が数分で読める1枚）へ組み立てるための写像規則。SKILL.md は手順と報告形式のみを持ち、「どの素材をどの順で・どの見出しに配置するか」は本ルールを参照する。

## 責務の境界（写像であって読み取り・照合ではない）

- 本ルールは **format 写像** である。素材の構成・順序・見出しへの割り当てだけを担う。
- 射影元（intent-tree / compass / packets / steering）を**読み直さない**。独自のパーサ・スキーマを持たず、source scope 解釈が受け渡した素材（どの層・どの見出し・どの packet 由来かを保ったもの）をそのまま並べ替える。
- 各記述のトレース付与・inferred 標識・不変則保持は**捏造抑制ルールの責務**であり、本ルールは行わない。canonical 由来の素材と inferred 由来の素材（Assumptions / Open Questions）は区別したまま受け取り、その区別を崩さずに配置する。

## 読み手と設計制約（結論先行・BLUF）

読み手はコードを書かない意思決定者（経営・営業・顧客窓口・非開発の個人開発者など）を想定する。この読み手は詳細より「結局これは何で、なぜ要るのか」を先に知りたい。したがって本 format は **結論先行（BLUF: Bottom Line Up Front）** を構成の核とする（DR93 の `bluf-message`）。

- **要点を冒頭に置く。** 何を作るか・なぜ作るかを最初の段落で言い切り、背景・経緯はその後に置く（背景から入って結論が末尾に来る構成にしない）。
- **1出力1目的（種別分離）。** 一枚ものは「何を・なぜ・現在地」を伝える文書に徹し、手順書・リファレンス・技術詳細を同じ出力に混ぜない（DR93 の `doc-type-separation`）。
- **読み手に内部記号を向けない（A33）。** `INV`・`DR`・`pkt-` などの内部識別子を裸で出さない。開き方は `rules/reader-vocabulary.md` に一本化する（記号 INV/DR は台帳 `scripts/symbol-labels.json` で言い換え・台帳に無い識別子〔packet id・スキル名等〕は省く）。本 format の読み手は既定で非当事者ゆえ、常にこの開きを適用する。
- **予定表・ベロシティを載せない（INV62）。** 日付コミットの予定や速度指標は出さず、現在地と前向きの順序までに留める。
- **組織の在否・実測の生データを載せない（INV60）。** 要約と出所参照までに留める。

## 構成（上から下へ：結論 → なぜ → やらないこと → 現在地）

source scope 解釈が受け渡した三層の素材を、次の順で配置する。

| 順 | セクション | 何を置くか | 主な素材の由来 |
|---|---|---|---|
| 1 | 結論（何を作るか） | この取り組みが何で、読み手にとっての価値は何かを最初に言い切る | Intent の North Star（compass）＋ intent-tree の L0（価値注記があれば添える） |
| 2 | なぜ作るか | 解こうとしている課題・目指す姿 | intent-tree の L0–L1（why の上位・計測基準があれば普通語で） |
| 3 | やらないこと | この取り組みで扱わない範囲（期待の先回り） | compass の Anti-direction ＋ packets の Non-scope の要約 |
| 4 | 今どこまで来たか | 現在地（群単位の状態要約。日付・速度は出さない） | packets の状態（index / active の state）を束ねた要約 |
| 5 | 前提・未確定（あれば） | 確定した上記と**混ぜずに**別枠で置く前提・未解決の問い | intent-tree の Assumptions / Open Questions（inferred 由来） |

- セクション 1（結論）を必ず冒頭に置く。背景（2）を先頭に出さない（結論先行の核）。
- 内部識別子は本文へ出す前に普通の言葉へ言い換える（言い換えられないものは省く）。
- 該当する素材が無いセクションは「未記入／該当なし」と明示して省く（推測で盛らない）。とくに価値注記・見積もりが無いときは、無いことを正直に出す。
- セクション 5 は inferred 由来を canonical の本文へ溶かさず末尾の別枠に置く（素材が無ければ省く）。

## 不変条件

- 射影元を読み直さない・変更しない（写像のみ）。
- 結論（何を・なぜ）を冒頭に置く順序を崩さない（BLUF）。
- 内部識別子（INV/DR/pkt-）を裸で読み手へ向けない（A33）。
- 予定表・ベロシティ・組織の在否・実測の生データを載せない（INV62 / INV60）。
- canonical 由来と inferred 由来を混在させない（標識は捏造抑制ルールに委ねる）。
- 素材に無い背景・動機・詳細を、文章を整えるために足さない。
