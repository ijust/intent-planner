# 引き継ぎブリーフ（handoff brief・指定があるときだけ生成する面）

セッションを乗り換えるとき、次のセッションが着手できる短い引き継ぎブリーフを `.intent/` の現在の状態から派生生成し、`.intent/handoff/` へ書き出す手順。`intent-overview` の Step 2 で、**利用者が引き継ぎブリーフを求めたときだけ**適用する（無指定の既定実行では本面を生成せず、Output の誘導1行だけを出す）。会話ログは読まず、トークン量も測らない（それはハーネスの領分・INV82・INV22）。ブリーフは派生であって正本ではなく、乗り換えの促しは案内までで強制しない（C-hb2・INV82・DR142・DR144）。

## 適用条件（既定では走らない）

- 利用者が `/intent-overview` に**引き継ぎブリーフを要求した**ときだけ生成する（自然言語トリガ「引き継ぎブリーフ」「handoff」「乗り換え」「次のセッションに渡したい」等）。既定の俯瞰ビューには自動同梱しない（pull 規律・コンテキストコスト）。
- 無指定の既定実行では、本面を生成せず overview.md の誘導1行に留める。既定実行の出力・所要は従来と変わらない（behavior-preserving）。
- 素材が薄い（active packet ゼロ・plan 未記入等）ときも停止せず、該当要素を「材料なし」と明示したブリーフを返す（エラーにしない・捏造しない・推測で埋めない）。

## 会話ログ・計測を読まない（層の分離・最重要）

- **会話履歴の要約・自動 compaction はハーネスの領分**であり、本面は触れない（Anti 433）。ブリーフの素材は `.intent/` に直列化済みの状態（下記の6要素）だけで、会話ログ・トークン数値を入力に取らない。
- 「文脈が長い／短い」の判断も本面はしない（それは促し側 = 完了報告の1行が定性で扱う）。本面は**求められたら生成する**だけ。

## 手順

1. **6要素の素材を `.intent/` から read-only で集める**（すべて読み取りのみ・手動プロトタイプ実証の6要素）:
   - **(1) 現在地（1分で掴む文脈）**: いま何の案件を・どの工程まで進めているか。`.intent/discovery/<スラッグ>-<rand>/mode.md`（引き継がれた発行ディレクトリ）・`.intent/packets/plan.md` の最新ジャーニー記録・直近の `.intent/deltas/` から、1〜2段落で「これまでの流れ」をまとめる。
   - **(2) 残タスク**: `.intent/packets/active/*.md` の frontmatter（`state`・`depends_on`）と、あれば `.intent/packets/plan.md` の工程計画節（推奨着手順）から、次に着手すべきもの・ブロック中のものを列挙する。
   - **(3) 正本への pull ポインタ（最小の読むリスト）**: 着手時に読むべき正本を**参照として**挙げる（該当 packet ファイル・compass の関係する記号〔INV/DR/Anti の番号〕・関係する rules）。**中身を全文転記しない**（Anti 434・pull 規律 INV47）。「まずこれだけ読めば着手できる」の最小リストにする。
   - **(4) 罠（Safety）**: 該当 packet の `## Safety / Invariants` から、踏むと事故る制約（守る約束・触ってはいけない境界）を要点だけ引く。
   - **(5) repo 状態**: 未コミットの変更・並行して動いている別セッションの注意。未コミットは git の状態（取得できなければ「未確認」と明示）、並行セッションは `.intent/assignments/` の割当宣言の実在ファイルからのみ読む（推測で「並行あり」と書かない）。
   - **(6) 実走オラクル**: 着手後に「効いたか」を確かめる手段（テストコマンド・実走の確認手順）。該当 packet の `## Validation` から引く。
2. **機微・ローカル事情の境界を守る（INV82-(3)）**: 詳細は正本への pull ポインタで示し、機微な内容を生のまま転写しない（定石 llm-sensitive-info-boundary）。ローカル事情（未コミット状態・並行割当・観点の在否）は本ブリーフが git 非追跡の `.intent/handoff/` に置かれるからこそ載せてよい（追跡される場所には載せない）。
3. **必須の注記を冒頭に置く**: 生成時点（ISO 8601・`date` で取得）と「**これは派生であって正本ではない・ブリーフだけで完結せず正本を pull する・食い違ったら正本が常に勝つ**」を明記する（INV82-(1)・Anti 438）。生成時点を取得できないときは推測の日付を書かず、その旨を明示する。
4. **`.intent/handoff/<ブリーフ名>.md` へ書き込む**: ファイル名は案件が分かる形（例: `handoff-<日付>-<案件slug>.md`）。書込みは `.intent/handoff/` 配下限定（canonical〔tree/compass/packets〕へ書かない・`.intent/overview/` にも書かない＝本面はブリーフ限定の明示例外の書き出し先を持つ）。同名があれば全置換（使い捨ての派生・再生成で直る）。

## 出力の形

- 冒頭: 生成時点・**派生であって正本ではない旨・正本が常に勝つ旨**・読んだ素材（active packets 件数・plan の有無等）。
- 本体: 6要素を見出しで分けて並べる — 現在地 / 残タスク / 正本への pull ポインタ / 罠（Safety）/ repo 状態 / 実走オラクル。各要素は出所を辿れる（捏造ゼロ・evidence-anchored と同線）。
- 素材が薄いとき: 該当要素を「材料なし」と明示する（無理に埋めない）。
- 末尾: 使い捨て・再生成可能・git 非追跡である旨の注記。

## 規律（守ること）

- **会話ログ・トークン計測を読まない（Anti 433・最重要のオラクル）**: 素材は `.intent/` の直列化済み状態のみ。会話履歴の要約・トークン数値を入力にしない（ハーネスの compaction と層が別）。
- **派生であって正本ではない（INV82-(1)）**: 生成時点と「正本が常に勝つ」注記を必ず置く。全文転記せずポインタ主体（Anti 434）。
- **機微を生転写しない（INV82-(3)）**: 詳細は正本への参照で示す。ローカル事情は git 非追跡の `.intent/handoff/` 限定で載せる。
- **canonical を書き換えない**: 書込みは `.intent/handoff/` のみ。packet 起票・優先度付け・Open Questions への書き戻しはしない（read-only mirror）。
- **出所を辿れないエントリを出さない**: 全要素が読んだ `.intent/` の素材へ辿れる（捏造ゼロ）。
- **促しを兼ねない**: 本面は生成まで。工程の切れ目での「続けるか乗り換えるか」の促しは完了報告側（intent-packets / intent-writeback）の役目（DR143）。
