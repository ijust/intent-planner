# Mermaid 図射影（mermaid views・指定があるときだけ生成する面）

意図の全体像と作業の順序・閉塞を「掴むための1枚の図」として Mermaid で描き、`.intent/overview/mermaid-views.md` へ派生出力する手順。`intent-overview` の Step 2 で、**利用者が図（マインドマップ／ロードマップ図）を求めたときだけ**適用する（無指定の既定実行では本面を生成せず、Output の誘導1行だけを出す）。文章の階層より図のほうが全体を早く掴めるという体験のための面であり、既定の tree 図 `mermaid-tree.md`（graph TD の詳細ビュー）とは目的が違う（C51/A59・DR116・INV62 の図適用）。

## 適用条件（既定では走らない）

- 利用者が `/intent-overview` に**図を要求した**ときだけ生成する（自然言語トリガ「マインドマップ」「図で見せて」「ロードマップを図で」「mindmap」等）。既定の俯瞰ビューには自動同梱しない（pull 規律・コンテキストコスト）。
- 無指定の既定実行では、本面を生成せず overview.md の誘導1行に留める。既定実行の出力・所要は従来と変わらない（behavior-preserving）。
- intent-tree が空／未生成、または packets ゼロのときは、その図を省略し理由を明示する（既存 rules の空表示規約と同じ・新規のエラー面を作らない・推測でノードを作らない）。

## 2つの図（1 rule に同居）

### 図1: 意図の全体像マインドマップ（intent tree の L0–L2 中心）

- **目的**: 「自分の考えの全体像を1枚で掴む」ための図。`mermaid-tree.md`（L0→L4 を graph TD で詳細に描く既定ビュー）とは別で、上位の意図（L0 目的 / L1 望む成果 / L2 能力）に絞って掴みやすさを優先する。
- **描画対象**: `intent-tree.md` の `## L0:`〜`## L2:` 各見出し配下の項目。掴むための図のため L3/L4 は既定で含めない（詳細は既定の tree 図に委ねる）。
- **記法の床（描画互換・最重要）**: GitHub / VSCode 標準で描画される記法に閉じる。Mermaid の `mindmap` 記法は描画対応が環境で割れるため、**既定は `graph TD` で擬似マインドマップ**（L0 を頂点に L1・L2 を放射状に結ぶ）を描く。`mindmap` 記法は、生成時に GitHub / VSCode の描画を実測して主要環境で描画されると確認できたときだけ使ってよい（DR116 Revisit when）。実験的記法・拡張図種・外部レンダラ・テーマ拡張を混ぜない（INV2・Anti 372）。
- **ノード ID / ラベル規約**: `mermaid-tree.md` の規約を流用する（ID は `L<レベル>_<インデックス>`・ラベルは `ID["..."]`・禁止文字 `( ) [ ] " /` は機械除去・長さ 40 で切り詰め・エッジは上位→下位）。ラベルに L 番号 / C 番号を残し、intent-tree の記述へ辿れるようにする（例1 の計測基準）。

### 図2: ロードマップ図（packet の順序と閉塞・日付なし）

- **目的**: 「何が何を塞いでいるか（作業の順序）」を図で掴む。`roadmap-projection.md`（テキストの前向き順序）の図版で、順序と閉塞まで（日付・進捗％・ガントを持ち込まない）。
- **描画対象**: `.intent/packets/active/*.md` の frontmatter（正本。`index.md` は再生成で直る派生キャッシュ）の `packet_id` / `name` / `state` / `depends_on`。
- **描き方**: `graph LR`（左→右）で、`depends_on` の「先に要るもの → それを待つもの」を `依存先 --> 依存元` のエッジで結ぶ。ノードのラベルに `state` を添えてよい（`state` は read-only で映すのみ・算出しない）。**依存先が done でない packet（＝まだ塞がれている）** が図で分かるようにする（例: ラベルに状態語を残す。色・クラス装飾は任意だが GitHub 描画の範囲に閉じる）。循環・未解決依存（active に無い packet への依存）はそのまま描く（潰さない）。
- **ラベルの禁止文字（図1と同じ床を図2にも適用）**: ノードラベルに `( ) [ ] " /` を含めない（`mermaid-tree.md` の規約）。`state` を添えるときは括弧でなく区切り（`— implementing` 等）を使う（`packet (implementing)` のような括弧付き注記は描画を壊す）。ID は `packet_id` から安定に導ける短縮キー（英数字と下線のみ）を使う。
- **日付なし（INV62・最重要のオラクル）**: 図に日付・締切・進捗％・ガント・ベロシティを一切出さない。順序（前後関係）と `state`・閉塞までに留める。

## 手順

1. 適用条件を満たすトリガのときだけ本面を生成する（既定では overview.md の誘導1行のみ）。
2. どの図を求められたかを読む（マインドマップ系／ロードマップ図／両方）。指定が曖昧なら両方を出してよい。
3. 図1は `intent-tree.md` の L0–L2 を、図2は active packet の frontmatter を read-only で読み、上記の記法の床で Mermaid を書き起こす。各図のすぐ後に対応するテキスト（図1は L0–L2 の階層・図2は順序と閉塞の箇条書き）を**正本として併記**する（図が描画されない環境でも通読が止まらないため・`mermaid-tree.md` の R3.2 と同線）。
4. `.intent/overview/mermaid-views.md` へ全置換で書き込む。冒頭に生成時点を明記する（derived・再生成可能・手動再生成のみ）。書込みは `.intent/overview/` 配下限定（canonical へ書かない）。

## 出力の形

- 冒頭: 生成時点・読んだ素材（intent-tree の L0–L2 の有無・active packet 件数）・**派生であり正本ではない旨**。
- 本体: 図1（マインドマップ系 Mermaid ＋テキスト階層）／図2（ロードマップ図 Mermaid ＋順序・閉塞のテキスト）。求められた図のみを出す。
- 空のとき: intent-tree 空なら図1を省略し理由を明示（`/intent-discover` を案内）。packets ゼロなら図2を省略し「packet なし」を明示。
- 末尾: 派生・正本ではない旨と、図が描画されない環境ではテキスト併記を読む旨の注記。

## 規律（守ること）

- **日付・進捗％・ガント・ベロシティを図に持ち込まない（INV62 の図適用・最重要のオラクル）**: ロードマップ図でも順序と `state`・閉塞まで。カレンダー・ガント・速度を図にしない（Anti 371）。
- **描画互換の床（Anti 372・L1-d 計測基準）**: GitHub / VSCode 標準で描画される記法に閉じる。`mindmap` 等の割れる記法は実測で確認できたときだけ・既定は `graph` 系。外部レンダラ・プラグイン・画像生成に依存しない（INV2）。
- **既定 tree 図（`mermaid-tree.md`）を本文改変しない**: 本面は別ビュー・別ファイルで、既存 tree 図の graph TD 規約・既定出力を byte 不変に保つ（pure addition）。
- **テキストを正本として常に併記する**: 図が壊れても通読が止まらないよう、各図に対応するテキストを添える（図の正しさをレンダラ実装に依存させない）。
- **`state` を算出・推論しない**: frontmatter の `state` を read-only で映すのみ。進捗を単一％に潰さない。
- **出所を辿れないノード・エッジを出さない**: 全ノードが intent-tree / packet frontmatter へ辿れる（捏造ゼロ）。
- **canonical を書き換えない**: 書込みは `.intent/overview/mermaid-views.md` のみ。図を見て着手順を実際に変えるのは人の判断（本面は射影まで）。
