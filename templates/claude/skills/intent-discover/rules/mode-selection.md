# Mode Selection

リポジトリの状況から Intent の詰め方モードを推奨するロジック。`/intent-discover` の冒頭で使う。これは拡張点でもある（新モードを足したら、ここに推奨条件を追記する）。

## 手順

1. **利用可能なモードを列挙する**
   - `.intent/modes/*.md` を Glob で探す。各モードの「適合する状況」節を読む。

2. **リポジトリ状況を軽く観察する**
   - 既存コードの有無・規模、テストの有無、README/docs の充実度、`.intent/` の既存成果物の有無。
   - 深く読み込まない。推奨に必要な手がかりだけ。

3. **モードを推奨する**
   - 現在の同梱モードは `standard` のみ。よって既定の推奨は `standard`。
   - 将来モードが増えた場合の推奨指針（このセクションを拡張する）:
     - 新規・意図が未言語化 → `standard`
     - 既存大規模・リファクタ → （`refactor` 系モードがあれば）それ
     - 振る舞いが不明なレガシー → （`behavior-unknown` 系モードがあれば）それ

4. **利用者に確認する**
   - 推奨モードと理由を提示し、`AskUserQuestion` で確認・変更を求める。
   - **モード候補が1つ（standard のみ）でも、推奨 → 確認 → 記録の配線は必ず通す**。これにより、将来モードが増えても利用者の体験が変わらない。

5. **確定結果を記録する**
   - 確定したモードを `.intent/mode.md` に書く（mode / selected / reason / definition）。

## 新モードを推奨対象に加える

新しい `.intent/modes/<name>.md` を作ったら、上記 3 の推奨指針に「どんなリポジトリ状況で <name> を推奨するか」を追記する。
